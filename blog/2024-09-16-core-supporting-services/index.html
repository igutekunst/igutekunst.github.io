Core Supporting Services

Published on September 16, 2024 | Last updated on September 20, 2024

#blog #development #infrastructure #ansible

In this post, I will describe the design of the core supporting services that will support the rest of the Personal Cloud.

Introduction

The core supporting services provide the foundation and support for building more advanced services, including Kubernetes clusters, load balancers, and more. They exist before any "user facing" services, and make deploying and managing the main services easier.

Goals

  1. Be able to quickly deploy the second layer of infrastructure
  2. Securely manage secrets
  3. Establish secure communication between all nodes, from bare metal physical servers, to ephemeral containers and virtual machines
  4. Observe what is happening in all services, including core support services, in the form of logs, tracing, error reporting and more
  5. Leave existing "legacy" infrastructure untouched until it is smoothly migrated

To achieve these goals, I will assemble the following tools into a system:

  • Use Packer for creating standardized VM images
  • Implement Terraform for infrastructure provisioning
  • Setup Headscale for secure networking
  • Set up Nomad for initial workload orchestration
  • Deploy Vault for secrets management
  • Set up monitoring tools (e.g., Prometheus, Grafana)
  • Implement logging solution (e.g., ELK Stack)

Legacy Considerations

Goal #5 has a strong influence on some design decisions. Notably, if I don't want to disturb existing legacy workloads, my entire system should have minimal affect on any running services. In practice, this means that I should perform the minimal amount of software and configuration changes on any existing nodes, and instead install a majority of the software into virtual machines that will serve as the first stable foundation layer.
